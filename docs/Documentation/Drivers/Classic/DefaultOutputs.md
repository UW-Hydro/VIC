    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-default-output-files" class="anchor" href="#default-output-files" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Default Output Files</h1>

<p>If the user does not give explicit output file instructions in the global parameter file, then VIC will produce a set of default output files, one per grid cell. The exact set of output files, and which variables they contain, is determined by model settings, as follows:</p>

<h2><a id="user-content-flux-files" class="anchor" href="#flux-files" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Flux Files</h2>

<p>The primary output file type for the VIC-NL model is the flux file, which contains information about moisture and energy fluxes for each time step. These output files are based on the model output files used for the PILPS-2C project. The number of variables in this file depends on the values of FULL_ENERGY and FROZEN_SOIL in the <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/GlobalParam.md">global parameter file</a>; when either FULL_ENERGY or FROZEN_SOIL are true, the file will contain the same number of columns as found in the PILPS-2C files for comparison purposes. Flux files are always written, regardless of the mode of operation. Flux files begin with the prefix "fluxes_".</p>

<p>For more information on the Flux Output Files, <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/FluxOutputFiles.md">click here</a>.</p>

<h2><a id="user-content-snow-files" class="anchor" href="#snow-files" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Snow Files</h2>

<p>The snow file contains information about the snowpack, averaged across all elevation bands and vegetation tiles. The set of variables in this file depends on the values of FULL_ENERGY and FROZEN_SOIL in the <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/GlobalParam.md">global parameter file</a>. Snow files are always written, regardless of the mode of operation. Snow files begin with the prefix "snow_".</p>

<p>For more information on the Snow Output File, <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/SnowOutputFile.md">click here</a>.</p>

<h2><a id="user-content-frozen-soil-files" class="anchor" href="#frozen-soil-files" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Frozen Soil Files</h2>

<p>When the model is run with FROZEN_SOIL set to TRUE in the <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/GlobalParam.md">global parameter file</a>, a third output file is produced which contains soil thermal output parameters. Frozen soil files begin with the prefix "fdepth_".</p>

<p>For more information on the Frozen Soil Output File, <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/FrozenSoilOutputFile.md">click here</a>.</p>

<h2><a id="user-content-snow-band-files" class="anchor" href="#snow-band-files" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Snow Band Files</h2>

<p>When the model is run with snow elevation bands, and PRT_SNOW_BAND is turned on in the <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/GlobalParam.md">global parameter file</a>, snow pack information for each elevation band will be output to files in the results directory with the prefix "snow_band_". Energy fluxes are output only for the full energy balance model, so there are file descriptions for both full energy and water balance model output files.</p>

<p>For more information on the Snow Band Output Files, <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/SnowBandOutputFiles.md">click here</a>.</p>

<h2><a id="user-content-lake-files" class="anchor" href="#lake-files" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Lake Files</h2>

<p>The lake file contains information about the lake fraction of the grid cell. Lake files are only written when LAKES is set equal to a valid lake parameter file, in the <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/GlobalParam.md">global parameter file</a>. Lake files begin with the prefix "lake_".</p>

<p>For more information on the Lake Output File, <a href="/UW-Hydro/VIC/blob/develop/docs/Documentation/Drivers/Classic/LakeOutputFile.md">click here</a>.</p>
